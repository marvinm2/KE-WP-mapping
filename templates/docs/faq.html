FAQ & Troubleshooting

Frequently Asked Questions

What is a Key Event?

A Key Event (KE) is a measurable biological change in an Adverse Outcome Pathway (AOP) that represents a critical step between a molecular initiating event and an adverse outcome. Key Events are documented in the AOP-Wiki database.

What is WikiPathways?

WikiPathways is a community-curated database of biological pathways. It contains pathway diagrams showing molecular interactions, gene networks, and biological processes across multiple species.

How do I choose the right confidence level?

Use the 4-question confidence assessment workflow. The system automatically calculates confidence based on:

  • Your knowledge of the connection (Evidence Basis)
  • How specific the pathway is to the Key Event (Pathway Specificity)
  • How completely the pathway captures the KE mechanism (KE Coverage)
  • The biological level of the Key Event (automatic weighting)

Why can't I submit mappings?

Mapping submission requires GitHub authentication. Click "Login with GitHub" in the top navigation, authorize the application, then you'll be able to submit new mappings and propose changes.

What does "Causative" vs "Responsive" mean?

  • Causative: The pathway causes or leads to the Key Event (Pathway → KE)
  • Responsive: The Key Event triggers or activates the pathway (KE → Pathway)
  • Bidirectional: Both directions apply
  • Unclear: Relationship exists but direction uncertain

Scoring & Pathway Suggestions

How are pathway suggestions generated?

The system uses a three-signal hybrid approach:

  • Gene-Based (35%): Finds pathways containing genes associated with the KE - provides unique biological validation
  • Text-Based (25%): Matches KE title terminology with pathway names
  • Semantic/BioBERT (40%): Neural network-based biological meaning similarity - highest weight due to best coverage and accuracy

See the Scoring Systems Guide for detailed documentation.

Why do some KEs get better suggestions than others?

Suggestion quality depends on:

  • Gene associations: KEs with many gene links get better gene-based matches
  • Biological level: Molecular/cellular KEs match pathways more directly than phenotypic KEs
  • Title specificity: Well-named KEs with biological terms match better

What does the confidence percentage mean?

Confidence scores (0-100%) indicate how strongly the system believes a pathway is relevant:

  • >75%: Strong match across multiple methods
  • 50-75%: Good match with some uncertainty
  • 25-50%: Moderate relevance, worth reviewing
  • <25%: Weak match, likely tangential

Why do BioBERT semantic scores seem lower now?

Semantic scores use a power transformation (score^1.5) to spread values more evenly. Raw BioBERT scores tend to cluster at 0.85-0.95 for all biomedical text. The transformation makes differences between good and marginal matches more visible:

  • 0.95 raw → 0.82 transformed (excellent match)
  • 0.85 raw → 0.62 transformed (good match)
  • 0.75 raw → 0.43 transformed (moderate match)

How is the confidence assessment calculated?

Your answers to the 4-question workflow are scored:

  • Evidence Basis: 0-3 points
  • Pathway Specificity: 0-2 points
  • KE Coverage: 0-1.5 points
  • Biological Level Bonus: +1 for molecular/cellular/tissue KEs

Total ≥5.0 = High confidence, ≥2.5 = Medium, <2.5 = Low. See the Confidence Assessment section for details.

Troubleshooting

Dropdown not loading / showing "Loading..."

Cause: SPARQL endpoint timeout or network issues

Solution:

  • Wait a few moments and refresh the page
  • Check your internet connection
  • AOP-Wiki or WikiPathways SPARQL endpoints may be temporarily unavailable

Pathway suggestions not appearing

Cause: No strong matches found or missing gene data

Solution:

  • Use the pathway search feature instead
  • Manually browse pathways from the dropdown
  • AI suggestions work best for molecular/cellular-level KEs with gene associations

SPARQL endpoint timeout errors

Cause: External SPARQL services taking too long to respond

Solution:

  • The application has built-in caching (24-hour expiry)
  • Retry the request - cached data will load faster
  • Report persistent issues on GitHub

GitHub login issues

Common causes:

  • OAuth app not properly configured (admin issue)
  • Callback URL mismatch in GitHub settings
  • Browser blocking third-party cookies

Solution: Contact the administrator or report the issue on GitHub

Known Limitations

Pathway search limitations (GitHub issue #53):

  • Search currently works by pathway title/description only
  • Ontology-based search not yet supported
  • Gene-based pathway search coming in future update
